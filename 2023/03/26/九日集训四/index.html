例题

1、 重新排列数组

思路分析


&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;这一题要关注的点如下：

  • 1、函数内申请的数组ret是跟传入函数的数组nums不同的。如果误以为两者是同一个数组，那么就无从下手；
  • 2、数组排列的方法，关键在于弄清楚同一个元素在排列前后的两个下标之间有什么关系。

    &#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;x的下标在排列之前原本为0~n-1，排列之后它从下标0开始隔项放入，对应的新下标都是偶数；y的原下标则是n~2n-1，排列之后它从下标1开始隔项放入，对应的新下彪都是奇数。联系各元素的原下标与新下标，可以得到下列关系：
    • 若新下标i为偶数，则对应的原下标j = i / 2;
    • 若新下标i为奇数，则对应的原下标j = n + i / 2
  • 3、 判断下标奇偶数的方法除了常用的对2取模，还可以用与运算。这涉及到二进制的与运算，这里不做展开。总之，以后判断奇偶数可以通过变量和1的与运算来判断变量的奇偶。

    i & 1为True，则i为奇数；i & 1为False，则i为偶数。

问题

为啥当i为偶数的时候，对应下标是(i + 1) / 2呢？虽然根据除法向下取整的特性倒也不影响结果，但是为什么不用i / 2呢？

4、前缀和

做法对比

  • 我的做法如下 就是按照常规的逻辑，动态和数组第i项元素的值等于数组前i项元素的和。所以这里用了循环嵌套。不说简洁，至少在不考虑时间复杂度的情况下是可以做出来的。

    但这里有一个错误，就是没有考虑第一个元素，sum[0]没有初始化，那么在循环里开始累加时就会报错。因此要顺利通过，还应该在循环外面添加一句sum[0] = nums[0];来初始化第一个元素。
  • 参考答案对比分析

    相比于我的做法，参考答案只用了一层循环便解决了问题。核心逻辑就是解析中的这句话:

    当 i = 0 时，动态和数组与nums数组元素一致，即 ret[0] = nums[0]； 当 i > 0 时，动态和数组第i项元素等于nums数组前i项元素的和，又等于动态和数组第i - 1项元素ms数组第i项元素，即 ret[i] = ret[i - 1] + nums[i]。

两者结合，使每一次循环都先赋值然后再累加，一层循环搞定。

6、 IP地址无效化

代码分析

  • 1、 申请一个新的字符串，长度定长一点。因为不必纠结于传入函数的字符串长度
  • 2、 两个字符串都从下标0开始遍历对比，如果没有遇到'.'字符，那么字符串就添加原字符，添加之后下标returnSize加1；如果遇到了'.'字符，那么先添加'['字符，下标returnSize加1，然后添加'.'字符，下标加1，再添加']'字符，下标加1.总而言之，下标returnSize始终指向下一格要添加字符的数组空间。
  • 3、 注意数组最后要加上字符串结尾符号'\0'。
  • 4、 字符串遍历的时候，循环中的循环条件表达式就不是整数型数组的i < numsSize 形式，而是直接用address[i]的形式，字符串对应元素不为空，则返回布尔值True，得以继续循环。

问题

遍历字符串的时候，空格会被视为空字符从而结束循环吗？'\0'字符呢？

7、 替换空格

思路分析

  • 这一题是和上一题IP地址无效化类似的字符串替换

代码分析

  • 1、 题目给定了字符串的长度，那么在申请新的字符串的时候，不能想当然地填10000个字节。而应该是300001个字节空间。其中30000是替换后字符串可能的最大值（10000个空格的字符串经过替换后长度变为30000），还有一个1是为了放入字符串结束符。
  • 2、 '%'符号作为c语言中的格式化运算符，如果要输出它自身则应该用'%%'，不能只用一个'%'。

8、 有多少小于当前数字的数字

思路分析

对于每个元素，都要统计数组中所有比它小的数字，因此对每个元素都要遍历一遍数组。所以需要两层循环嵌套。

代码对比

  • 我的做法如下 核心逻辑跟参考代码是一致的，都是双层循环嵌套。而我相比于参考代码，多加了一个变量sum用来统计每一次遍历时的统计结果。这本来无可厚非，但我的一个致命错误就是变量仅仅在循环外初始化了一次，导致最终sum的结果不是每个元素比它小的数字的数目，而是这些数目的累加和……
  • 参考代码分析

    &#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;代码没有多引入一个变量，而是在第一层循环将ret[i]初始化为0，然后再在下一层循环中更新。数组也可以++。

9、 打印1到最大的n位数

思路分析

  • 1、 弄清楚各个位数的数字个数，n位数总共有$2^{10}$ - 1个
  • 2、 弄清了n位数的数字个数就弄清楚了返回数组的长度

代码分析

  • 1、 几位数就是10的几次方减1，得出的结果就是数组长度。
  • 2、 这次的i从1开始，既做下标，也做数组元素，但要注意有偏移，数组元素无论如何都是从0开始算起。
  • 3、 不要先急着申请数组内存空间，先得到它的容量再申请。

10、按既定顺序创建目标数组

思路分析

&#160;&#160;&#160;&#160;&#160;&#160;&#160;&#160;运用了数组插入的算法

代码分析

  • 总体上的逻辑就是嵌套循环，运用了数组插入的算法。不过申请数组内存空间的时候不太明白为什么要跟nums数组容量一致而不是跟index数组。
  • (1) 这两个变量的定义是为了让代码更易读一些，即便删掉这两个变量，然后将它们全部用index[i]和nums[i]替代也是一样的。idx和ins分别表示当次循坏中要插入的下标位置和要插入的元素。
  • (2) 这一层循环就是数组插入算法中将数组待插入元素之后的元素后移的部分。有几个要点：
    1. 顺序是逆序，这是因为如果是正序的话，第一个元素后移一位，那么第二个元素就会被挤掉，到最后的结果就是虽然元素全部后移一位，但是值全部变成了第一个元素。
    2. 逆序的下标从当前数组长度len开始。在插入之前，数组长度为len，而最后一位元素的下标应该是len-1。插入1个元素之后，该元素之后的元素都要往后移一位，那么最后一位元素的下标就变成了（len-1）+1=len（这里的len表示的是插入之前的数组长度，还没更新）。
    3. 如果元素插入的位置为下标k，那么该元素后移之后的第一个元素的下标就为k+1。综上，可以得到元素后移的循环表达式for(j = length; j > k; j-- )
    4. ret[i] = ret[i-1]，元素后移。不需要考虑j=0时的情况，循环条件表达式已经解决了这个问题。
  • (3) 元素完成后移，给待插入的元素腾出了空间，接下来就是把元素插入对应位置了。
  • (4) 插入之后，数组长度加1
